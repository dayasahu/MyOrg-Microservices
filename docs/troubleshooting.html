Troubleshooting & FAQ

The real problems you'll hit running this platform locally — each with the cause and the exact fix.

Networking & Services

host.docker.internal connection refused from a pod

Cause: host.docker.internal resolves from Docker containers but not from inside Kubernetes pods.

Fix: use Kubernetes DNS (alloy.observability.svc.cluster.local) or the real machine IP — never host.docker.internal in pod env vars.

Service exists but returns 502 / times out — endpoints <none>

Cause: the Service selector doesn't match any pod labels. With Argo Rollouts this often happens when the active Service still carries an old rollouts-pod-template-hash.

kubectl get endpoints employee -n microservices    # <none> = no matching pods
kubectl get pod <pod> --show-labels -n microservices
kubectl get svc employee -o jsonpath="{.spec.selector}" -n microservices

Fix: let Argo manage it — kubectl argo rollouts promote employee re-points the active Service at the current ReplicaSet. Don't hand-edit the selector.

Deployments & Rollouts

"deployments.apps employee not found"

Cause: employee and department are Argo Rollouts, not Deployments. kubectl set image deployment/employee / rollout status deployment/employee won't find them.

Fix: use the rollouts plugin:

kubectl argo rollouts set image employee employee=user/img:tag -n microservices
kubectl argo rollouts status employee -n microservices

Rollout stuck — status Paused / BlueGreenPause

Cause: this is expected. With autoPromotionEnabled: false the green version waits for a manual promotion after the smoke analysis passes.

kubectl argo rollouts get rollout employee -n microservices --watch
kubectl argo rollouts promote employee -n microservices   # flip green → active

Old pods keep coming back / multiple versions running

Cause: leftover ReplicaSets from a controller switch (e.g. a Deployment and a Rollout both managing employee).

Fix: use only one controller per workload. Delete the stale Deployment, let the Rollout own the pods, then remove orphaned ReplicaSets:

kubectl delete deployment employee -n microservices --ignore-not-found
kubectl get rs -n microservices -l app=employee

Images

403 denied on docker push

Cause: the images target dayasahu6077/* — you don't have write access.

Fix: retag to your own Docker Hub namespace, or skip push and use local images. See Docker Hub & Registry Login.

Code changed but the pod runs the old image

Cause: imagePullPolicy: Always forces a registry pull, ignoring your freshly built local image.

Fix: for local iteration set imagePullPolicy: IfNotPresent on the workload so it uses the local image.

Observability

make obs-up hangs / "deployments.apps otel-collector not found"

Cause: the OTel Collector was replaced by the Grafana Alloy DaemonSet. An old target waited for the now-deleted deployment.

Fix: already corrected — obs-up now waits for daemonset/alloy. If you see this, pull the latest Makefile.

No logs in Loki / Grafana Explore is empty

Cause: Alloy needs RBAC to read pod logs, or it started before the services.

kubectl logs daemonset/alloy -n observability --tail=20      # look for "forbidden: pods/log"
kubectl exec deploy/loki -n observability -- wget -qO- http://localhost:3100/loki/api/v1/labels

Fix: ensure the Alloy ClusterRole grants pods/log, then kubectl rollout restart daemonset/alloy -n observability.

Tempo "connection refused" right after startup

Cause / Fix: transient — Tempo's gRPC listener starts after HTTP. It resolves on its own within a few seconds; no action needed.

Prometheus "out of order sample"

Cause / Fix: transient after a restart when duplicate samples arrive. Self-heals in a few minutes.

Auth & Config

401 Unauthorized through the gateway

Cause: missing or invalid Authorization: Bearer token. Only /auth/** is whitelisted.

Fix: get a token from POST /auth/token and send it on every other call. Token expiry is 24h.

Service starts but config values are missing

Cause: the Config Server serves properties keyed by SPRING_APPLICATION_NAME. If it doesn't match the .properties filename, nothing is found.

kubectl exec <pod> -n microservices -- env | grep SPRING_APPLICATION_NAME
# must match the file basename, e.g. employee → employee.properties

Kubernetes Dashboard

Browser warns about an invalid certificate

Cause / Fix: the dashboard uses a self-signed cert. Expected on https://localhost:30443 — proceed past the warning.

Login needs a token

kubectl -n kubernetes-dashboard create token admin-user   # short-lived
# static token: see Setup → Kubernetes Dashboard + Static Token

Quick FAQ

Do I need Maven/JDK?
No — the Dockerfile builds the jar. Only needed for building outside Docker (mvnw.cmd).
Do I need to push images?
Not for local dev — Docker Desktop's K8s sees local images (set imagePullPolicy: IfNotPresent).
Where are the gateway routes?
config-map-k8/apigateway/base/apigateway.properties.
Why two namespaces?
microservices for apps, observability for the LGTM stack — clean blast-radius separation.
Is the data persistent?
No — services use in-memory H2 and observability uses emptyDir. Restarts lose data (fine for a training setup).
