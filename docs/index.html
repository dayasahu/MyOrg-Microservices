MyOrg Microservices Platform

A production-grade Spring Boot microservices system running on Kubernetes — complete with an API gateway, centralized configuration, full LGTM observability, and zero-downtime blue-green deployments driven by GitHub Actions and Argo Rollouts.

Java 21 Spring Boot 3.5.8 Spring Cloud 2025.0.0 Kubernetes Argo Rollouts Grafana LGTM GitHub Actions

This documentation explains every layer of the platform — from a single API request travelling through the gateway, down to how metrics, logs and traces are collected and visualised. It is written so that anyone cloning the repo can understand the design decisions and stand the whole system up locally on a Windows machine.

What's Inside

⊟

API Gateway

Spring Cloud Gateway routes all traffic and validates JWTs in one place.

⛁

Config Server

Centralized Spring Cloud Config in native mode, backed by Kubernetes ConfigMaps.

🔐

JWT Auth

Dedicated auth-service issues HS256 tokens; the gateway enforces them.

📊

LGTM Observability

Loki, Grafana, Tempo, Prometheus + Grafana Alloy as a single collector.

🚀

Blue-Green

Argo Rollouts with automated smoke analysis and instant rollback.

∞

CI/CD

GitHub Actions pipeline: build → test → deploy green → approve → promote.

End-to-End Architecture

Figure 1 — End-to-end request flow and telemetry pipeline across the two namespaces.

A Request, Step by Step

Client hits the ingress

A request to http://api.myorg.com/employee/api/employees resolves (via the hosts file locally) to the NGINX ingress controller.

Ingress forwards to the gateway

The ingress sends everything under / to the apigateway service on port 80.

Gateway validates the JWT

The global JwtAuthenticationFilter checks the Authorization: Bearer header. /auth/** is whitelisted; everything else needs a valid token.

Gateway routes & strips the prefix

/employee/** → http://employee:80 with StripPrefix=1, so the employee service receives /api/employees.

Service-to-service call

The employee service uses a reactive WebClient to call http://department/api/list for the departments endpoint.

Telemetry streams out

Throughout, each service emits traces & metrics over OTLP to Grafana Alloy, while Alloy also tails pod stdout logs — all forwarded to Tempo, Prometheus and Loki.
