Docker & Kubernetes Cheat Sheet

The commands worth memorising — grouped by task, with the project-specific ones called out so you can drive this repo without hunting through docs.

Set a default namespace so you can drop -n microservices from every command: kubectl config set-context --current --namespace=microservices

🐳 Docker

Images

docker build -t user/app:tag .          # build from Dockerfile in current dir
docker images                           # list local images
docker tag user/app:tag user/app:latest # re-tag
docker rmi user/app:tag                 # remove an image
docker pull user/app:tag                # pull from registry
docker push user/app:tag                # push (needs docker login)
docker login                            # authenticate to Docker Hub

Containers

docker ps                               # running containers
docker ps -a                            # all (incl. stopped)
docker run --rm -p 8080:8080 user/app:tag # run & auto-remove on exit
docker exec -it <container> sh          # shell into a container
docker logs -f <container>              # follow logs
docker stop <container> ; docker rm <container>

Cleanup (reclaim disk)

docker system df                        # what's using space
docker image prune                      # dangling images
docker system prune -a                  # EVERYTHING unused (careful)
docker builder prune                    # build cache
Project build/push: handled by make build / make push. Push targets the dayasahu6077/* namespace — see Docker Hub & Registry Login to use your own account.

☸️ kubectl — Everyday

Context & cluster

kubectl config get-contexts             # list contexts
kubectl config use-context docker-desktop
kubectl cluster-info
kubectl get nodes -o wide

Inspecting resources

kubectl get pods -n microservices               # list pods
kubectl get pods -A                             # all namespaces
kubectl get pods -o wide                         # + node & IP
kubectl get all -n microservices                # pods/svc/deploy/rs
kubectl get svc,ingress -n microservices
kubectl describe pod <pod> -n microservices      # events & details
kubectl get pod <pod> -o yaml                    # full manifest
kubectl get events -n microservices --sort-by=.lastTimestamp

Logs & exec

kubectl logs -f <pod> -n microservices                  # follow logs
kubectl logs <pod> --previous                           # crashed container's logs
kubectl logs -l app=employee --all-containers -n microservices
kubectl exec -it <pod> -n microservices -- sh          # shell in
kubectl exec <pod> -n microservices -- env             # dump env vars

Apply / delete

kubectl apply -f manifest.yaml          # single file
kubectl apply -k .                      # Kustomize dir (this repo's root)
kubectl delete -k .                     # delete what Kustomize created
kubectl delete pod <pod> -n microservices               # pod is recreated by its controller
kubectl rollout restart deployment/employee -n microservices

Workloads & Scaling

kubectl get deploy -n microservices
kubectl rollout status deployment/apigateway -n microservices
kubectl rollout history deployment/apigateway -n microservices
kubectl rollout undo deployment/apigateway -n microservices    # roll back a Deployment
kubectl scale deployment/apigateway --replicas=3 -n microservices
kubectl set image deployment/apigateway apigateway=user/img:tag -n microservices
kubectl autoscale deployment/employee --min=2 --max=8 --cpu-percent=70  # HPA
employee & department are Argo Rollouts, not Deployments — the kubectl rollout … deployment/… commands above do not apply to them. Use the Argo Rollouts commands below.

🚀 Argo Rollouts (blue-green)

kubectl argo rollouts list rollouts -n microservices
kubectl argo rollouts get rollout employee -n microservices --watch   # live view
kubectl argo rollouts set image employee employee=user/img:tag -n microservices
kubectl argo rollouts promote employee -n microservices               # green → active
kubectl argo rollouts status employee -n microservices --timeout=180s
kubectl argo rollouts abort employee -n microservices                 # stop a bad rollout
kubectl argo rollouts undo employee -n microservices                  # roll back to blue

ConfigMaps & Secrets

kubectl get configmap -n microservices
kubectl describe configmap employee-config -n microservices
kubectl get secret jwt-secret -n microservices -o jsonpath="{.data.JWT_SECRET}" | base64 -d  # decode
kubectl create secret generic my-secret --from-literal=KEY=value -n microservices
kubectl create configmap my-cm --from-file=app.properties -n microservices

Networking & Access

kubectl get ingress -A
kubectl get svc -n observability                          # find NodePorts
kubectl port-forward svc/grafana 3000:3000 -n observability   # tunnel a ClusterIP service
kubectl get endpoints employee -n microservices           # is the service wired to pods?
# DNS test from inside a pod:
kubectl exec <pod> -n microservices -- wget -qO- http://department/api/list
Empty endpoints? If kubectl get endpoints <svc> shows <none>, the Service selector doesn't match any pod labels — the #1 cause of "service exists but 502/timeout".

Debugging Playbook

SymptomCommand to run
Pod won't start / CrashLoopBackOffkubectl logs <pod> --previous + kubectl describe pod <pod>
Pending podkubectl describe pod <pod> → check Events (resources / scheduling)
Service returns errorskubectl get endpoints <svc> → must list pod IPs
Config not appliedkubectl exec <pod> -- env → verify env vars
Image not updatingcheck imagePullPolicy; for local images use IfNotPresent
Rollout stuckkubectl argo rollouts get rollout <name> --watch

Kustomize

kubectl kustomize .                     # render manifests without applying
kubectl apply -k .                      # build + apply
kubectl apply -k observability/k8       # the observability stack
kubectl diff -k .                       # preview changes vs cluster
