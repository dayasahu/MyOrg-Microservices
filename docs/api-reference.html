API Reference

Every endpoint each service exposes, how to reach it through the gateway, and the actuator paths used for health and metrics.

Everything goes through the gateway. Externally you call http://api.myorg.com/<service>/.... The gateway strips the first path segment (StripPrefix=1), so /employee/api/employees hits the employee service as /api/employees. All routes except /auth/** require a Bearer JWT.

auth-service

Issues JWTs. Reached via /auth/** (whitelisted — no token required).

MethodGateway pathService pathBody / Result
POST/auth/token/tokenBody {"userId":"12345"} → {"token":"<jwt>"}
GET—/swagger-ui/index.htmlOpenAPI UI (direct on the service)
 
# Get a token
curl -X POST http://api.myorg.com/auth/token \
  -H "Content-Type: application/json" \
  -d "{\"userId\":\"12345\"}"
# → {"token":"eyJhbGciOiJIUzI1NiJ9..."}

employee

Base path /api. Reached via /employee/**.

MethodGateway pathReturns
GET/employee/api/hello"Welcome to Demo"
GET/employee/api/employeesJSON array of 5 employees (id, name, role)
GET/employee/api/profileActive Spring profiles, e.g. ["dev"]
GET/employee/api/envmsgMessage from config (poc.environment.msg)
GET/employee/api/departmentsCalls department (http://department/api/list) reactively and returns its response
 
TOKEN=$(curl -s -X POST http://api.myorg.com/auth/token \
  -H "Content-Type: application/json" -d "{\"userId\":\"12345\"}" | jq -r .token)

curl http://api.myorg.com/employee/api/employees   -H "Authorization: Bearer $TOKEN"
curl http://api.myorg.com/employee/api/departments -H "Authorization: Bearer $TOKEN"  # cross-service

Actuator (all services)

Every service exposes Spring Boot Actuator. These are used by Kubernetes probes and Prometheus — and are handy for debugging.

PathPurpose
/actuator/healthOverall health (details shown)
/actuator/health/livenessK8s liveness probe — restart if down
/actuator/health/readinessK8s readiness probe — also drives blue-green smoke analysis
/actuator/prometheusMetrics in Prometheus format (scraped)
/actuator/metricsBrowse individual metrics
/actuator/info · /env · /loggers · /mappingsBuild info, resolved config, log levels, route mappings
 
# From inside the cluster (or via port-forward):
kubectl exec deploy/apigateway -n microservices -- wget -qO- http://localhost:8888/actuator/health
# Readiness used by the blue-green smoke test:
.../actuator/health/readiness   # expects {"status":"UP"}
